Ideias propostas:
Cadastro de novos alunos
Consulta rápida
Atualização de informações
Relatórios básicos (como alunos ativos, busca por nome, etc.)

Requisitos Funcionais:
Permitir cadastrar um novo aluno com seus dados básicos.
Exibir uma lista de todos os alunos cadastrados.
Permitir buscar alunos pelo nome, matrícula ou outro identificador.
Atualizar informações existentes já no cadastro.
Exibir relatórios simples como: alunos ativos, total de alunos, busca filtrada

Requisitos Não Funcionais:
O sistema deve ser fácil de usar e interface intuitiva.
O código deve ser modular.
O sistema deve funcionar em qualquer computador com Python instalado.
Todas as operações devem ser rápidas.
A interface gráfica será feita com Tkinter.

Regras de negócio:
Todos os campos devem ser preenchidos obrigatoriamente antes de salvar aluno.
Não pode haver dois alunos com a mesma matrícula.
Um aluno deve ter um status (ativo/inativo).
O nome não pode conter números.
